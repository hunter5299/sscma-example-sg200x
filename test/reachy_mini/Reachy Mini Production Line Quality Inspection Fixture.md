# Reachy Mini Production Line Quality Inspection Fixture

## Introduction

reCamera 是一款具备边缘计算能力的 AI 摄像头，支持 ROI、OpenCV 图像处理、OSD 叠加与 YOLO 推理等能力。本项目面向 Reachy Mini 产线舵机装配场景，目标是将原先依赖人工目检的质检步骤升级为自动化在线检测治具。

本治具方案聚焦两类关键质量点：

- 凹槽方向检测：通过凹槽数量和形态判断舵机臂是否装反。
- 字符与编码检测：通过 OCR 读取并校验关键标识，避免错料与漏标。

完成部署后，产线可实现以下能力：

- 实时判定：单件检测结果在工站节拍内返回。
- 结果可追溯：每件产品保留结构化检测记录。
- 标准统一：同一规则覆盖不同班次与不同操作员。
- 易扩展：后续可叠加更多视觉规则（尺寸、缺陷、装配状态）。


## Hardware Preparation

- 一台 reCamera（本项目使用 reCamera HQ PoE）
- 一台上位机（用于部署、调试与结果接收）
- Reachy Mini 舵机与稳定光源
- 局域网连接（设备与上位机在同一网段）

现场配置：

- 光源：使用固定角度条形光或环形光，避免工件表面强反光。
- 安装：相机、工件定位块、光源三者做刚性固定，避免位置漂移。
- 网络：设备与上位机使用有线网络，降低丢包和延迟波动。
- 供电：PoE 或稳定电源，避免供电波动导致重启。

## 1. 项目范围与产线目标

### 1.1 适用工位

本方案适用于 Reachy Mini 舵机装配后的在线质检工位，重点覆盖：

- 舵机臂方向正反检查
- 产品标识可读性与规则一致性检查

### 1.2 目标指标

- 漏检率（NG 被误判为 OK）：小于 0.005%
- 误检率（OK 被误判为 NG）：小于 0.005%
- 单件检测时延：小于 200 ms（不含人工取放件）
- 连续运行稳定性：7x24 小时无异常崩溃

### 1.3 判定输出定义

- OK：允许流转到下工序
- NG：拦截并返工
- RECHECK：图像质量不足或规则冲突，进入人工复检

## 2. 项目目标与判定标准

### 2.1 业务背景

在 Reachy Mini 舵机装配工位，历史流程依赖人工观察舵机臂凹槽方向来判断正反，存在如下问题：

- 人工疲劳导致漏检与误判
- 班次间判定标准不一致
- 质检记录难以结构化沉淀

引入视觉治具后，系统以统一规则执行检测，并且每次检测输出同一格式的结果数据，为后续质量分析和工艺优化提供依据。

### 2.2 判定规则

本项目采用统一规则进行自动判定：

- 检测到两条有效凹槽：判定为正向（OK），允许流转。
- 检测到一条有效凹槽：判定为反向（NG），触发返工。
- 凹槽数量异常或图像质量不达标：判定为待复检（RECHECK）。

补充规则：

- 若 OCR 识别置信度低于阈值（例如 0.80），强制 RECHECK。
- 若同一帧中凹槽检测与 OCR 所属型号规则冲突，判定 NG。
- 连续 N 帧（例如 3 帧）一致后再出最终判定，降低抖动误判。

### 2.3 系统架构与数据流

项目数据流如下：

1. 相机采集工件图像。
2. 凹槽检测模块输出凹槽数量与定位框。
3. OCR 模块输出文本、置信度、规则匹配结果。
4. 判定逻辑对三项结果分别输出红绿状态（舵机是否装反、型号、小黑点）。
5. 结果通过 UDP/MQTT 发送到上位机与产线系统。
6. 上位机记录日志、生成报表并展示实时画面。

数据字段统一为：

- product_id
- station_id
- timestamp
- groove_count
- groove_score
- ocr_text
- ocr_score
- final_result
- reason_code
- image_path（可选）

## 3. 凹槽检测方案（OpenCV + ROI）

### 3.1 方案说明

凹槽检测基于固定 ROI 内的传统图像处理流程，参考工程：

- solutions/sesg-project/groove_features
- GitHub: https://github.com/RobotXTeam/sscma-example-sg200x/tree/main/solutions/sesg-project/groove_features

核心流程：

1. 采集相机帧并裁剪固定 ROI。
2. 灰度化与阈值二值化，突出凹槽结构。
3. 形态学开闭运算，抑制噪声并连通目标。
4. 轮廓筛选与长宽比过滤，统计有效凹槽数量。
5. 输出凹槽计数与 bbox，可通过 UDP 实时发送给上位机。



### 3.2 部署步骤

1. 在设备端部署 groove_features 可执行程序。
2. 根据治具固定位置标定 ROI（避免覆盖背景干扰区域）。
3. 调整阈值与核大小，确保不同批次表面反光下仍可稳定检测。
4. 启动 UDP 推流，将检测结果回传至上位机用于可视化与记录。

部署完成后的现场自检：

- 连续拍摄 50 件 OK 样本，确认无误拦。
- 连续拍摄 50 件 NG 样本，确认可稳定拦截。
- 随机调整放置角度，验证对轻微姿态变化的鲁棒性。


## 4. OCR 检测方案（字符与编码校验）

### 4.1 方案说明

OCR 检测用于读取指定区域的字符或编码信息，参考工程：

- solutions/cosg-project/ppocr-reader
- GitHub: https://github.com/RobotXTeam/sscma-example-sg200x/tree/main/solutions/cosg-project/ppocr-reader

典型用途：

- 检查批次号、序列号、型号编码是否存在且清晰。
- 识别结果与工单规则比对，输出 PASS/FAIL。

OCR 关键注意点：

- 字符区域应尽量正对相机，减少透视畸变。
- 优先保证字符边缘清晰，不要盲目提高曝光。
- 对固定格式文本可加入正则规则，例如字母+数字长度约束。

### 4.2 部署步骤

1. 在治具中固定文字区域拍摄角度和工作距离。
2. 设置 OCR ROI，优先保证字符高度与对比度。
3. 根据现场字体、打码深浅和材质反光调优预处理。
4. 输出结构化结果（文本、置信度、规则匹配状态）供产线系统使用。

输出结构：

- text_raw：原始识别文本
- text_norm：归一化文本（去空格、统一大小写）
- confidence：OCR 置信度
- rule_match：是否命中工单规则
- fail_reason：失败原因（低置信度/格式错误/不在白名单）

### 4.3 与凹槽检测联动

项目上线逻辑与 solutions/vision/factory/feature/factory.py 保持一致，界面状态不是 PASS/NG/RECHECK 三态，而是三行独立红绿状态：

- 第 1 行「舵机是否装反」：groove_det_count == 2 显示绿色，否则显示红色。
- 第 2 行「型号」：若左路或右路连续 30 帧无文字，直接显示红色。
- 第 2 行「型号」：若连续无文字条件未触发，则进入稳定化联动校验（StabilizedLogicEvaluator），满足规则显示绿色，否则红色。
- 第 3 行「小黑点」：left_dot_exists 为 True 显示绿色，否则显示红色。

型号联动规则由 evaluate_linked_logic() 实现：

- 左路识别到 F，右路必须包含 XC330。
- 左路识别到 1-7 任一数字，右路必须同时包含 XL330 与 M288。
- 左路同时识别到 R 和 L，右路必须包含 M077。
- 若左路未触发以上任一条件，联动校验返回无需检查。

上位机侧实现了“判定解释”字段，便于现场快速定位问题，例如：

- E101：凹槽数量异常
- E201：OCR 低置信度
- E202：OCR 文本与工单不匹配
- E301：图像过曝或欠曝

## 6. 项目结论

通过在 Reachy Mini 产线部署 reCamera 质检治具，关键质检环节已从人工经验判断升级为标准化、可量化、可追溯的自动检测流程。该方案兼顾成本、部署效率与扩展性，支持从单工位验证逐步扩展到整线应用。

从实施经验看，项目成功的关键不是单一算法精度，而是“光学条件 + 工装稳定性 + 参数版本管理 + 现场闭环流程”四者协同。项目先在单工位形成稳定模板，再复制到同类产线，实现了低风险、可复用的规模化部署。
